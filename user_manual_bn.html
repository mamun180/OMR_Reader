OptiMark Pro ব্যবহারকারী ম্যানুয়াল

ভূমিকা

OptiMark Pro-তে আপনাকে স্বাগতম। এটি একটি OMR (Optical Mark Recognition) সমাধান যা OMR উত্তরপত্র তৈরি, স্ক্যান এবং মূল্যায়ন প্রক্রিয়াকে সহজ করার জন্য ডিজাইন করা হয়েছে। এই ম্যানুয়ালটি সফ্টওয়্যারের প্রাথমিক সেটআপ থেকে শুরু করে উন্নত ডেটা প্রসেসিং পর্যন্ত সমস্ত বৈশিষ্ট্যের জন্য একটি বিস্তারিত নির্দেশিকা প্রদান করে।

রেজিস্ট্রেশন

রেজিস্ট্রেশন মডিউলটি আপনার সফ্টওয়্যার লাইসেন্স সক্রিয়করণ পরিচালনা করে। অ্যাপ্লিকেশনটির সম্পূর্ণ কার্যকারিতা আনলক করার জন্য একটি বৈধ লাইসেন্স প্রয়োজন।

কার্যকারিতা ও কর্মপ্রবাহ

  1. OptiMark Pro চালু করুন। লাইসেন্স অবৈধ হলে, বেশিরভাগ বৈশিষ্ট্য নিষ্ক্রিয় থাকবে।
  2. Registration ট্যাবে নেভিগেট করুন।
  3. ইনপুট ক্ষেত্রে আপনার প্রাপ্ত লাইসেন্স কী লিখুন।
  4. Activate বোতামে ক্লিক করুন। অ্যাপ্লিকেশনটি আপনার কী যাচাই করতে এবং এটিকে আপনার মেশিনের অনন্য হার্ডওয়্যার আইডিতে লক করতে লাইসেন্স সার্ভারের সাথে সংযোগ স্থাপন করবে।
  5. সফল হলে, সমস্ত অ্যাপ্লিকেশন বৈশিষ্ট্য অবিলম্বে আনলক হয়ে যাবে।

দ্রষ্টব্য: লাইসেন্স সক্রিয়করণের জন্য একটি সক্রিয় ইন্টারনেট সংযোগ প্রয়োজন। লাইসেন্সটি একটি একক মেশিনের সাথে আবদ্ধ থাকে।

সেটিংস

সেটিংস পৃষ্ঠাটি সম্পূর্ণ অ্যাপ্লিকেশন কনফিগার করার কেন্দ্রীয় হাব। অন্য সমস্ত মডিউল সঠিকভাবে কাজ করার জন্য এখানে সঠিক সেটআপ অত্যন্ত গুরুত্বপূর্ণ।

ডিফল্ট পাথ

এই বিভাগটি অ্যাপ্লিকেশনটির জন্য মূল ফাইলের অবস্থানগুলি সংজ্ঞায়িত করে।

  • Base Data Directory: প্রধান ফোল্ডার যেখানে অ্যাপ্লিকেশন দ্বারা তৈরি সমস্ত ডেটা (টেমপ্লেট, উত্তর কী, ফলাফল) ডিফল্টরূপে সংরক্ষণ করা হবে।
  • Student Data Sheet: ছাত্রের তথ্য সম্বলিত একটি এক্সেল ফাইল। যখন আপনি একটি উত্তরপত্র স্ক্যান করেন, অ্যাপ্লিকেশনটি এই ফাইলটি ব্যবহার করে একজন ছাত্রের বিবরণ খুঁজে বের করে।
  • Master Template: উত্তর কী স্ক্যানার দ্বারা ব্যবহৃত ডিফল্ট OMR টেমপ্লেট (`.json` ফাইল)। এটি একটি গুরুত্বপূর্ণ সেটিং, কারণ স্ক্যানারটির স্ক্যান করা উত্তর কী শীটের লেআউট জানার জন্য একটি টেমপ্লেট প্রয়োজন।

ফাইলের নামকরণ এবং আউটপুট প্যাটার্ন

এই বৈশিষ্ট্যটি আপনাকে আপনার আউটপুট ফাইলগুলির নামকরণ এবং সংগঠন স্বয়ংক্রিয় করতে দেয়।

উদাহরণ:

ধরুন আপনি চান আপনার চূড়ান্ত এক্সেল রিপোর্টগুলির নাম হোক Results_Class_IX_Physics.xlsx। "Setup" ডায়ালগে, আপনি "Excel Filename" এর জন্য একটি প্যাটার্ন তৈরি করতে পারেন এইভাবে:

  1. একটি স্ট্যাটিক টেক্সট উপাদান যোগ করুন: "Results"
  2. Class নামের ROI-এর জন্য একটি উপাদান যোগ করুন।
  3. Subject নামের ROI-এর জন্য একটি উপাদান যোগ করুন।

অ্যাপ্লিকেশনটি এখন স্বয়ংক্রিয়ভাবে শীটের "Class" এবং "Subject" ROI থেকে স্ক্যান করা মানগুলি ব্যবহার করে এই নির্দিষ্ট ফাইলের নামটি তৈরি করবে।

অ্যাডভান্সড স্টুডেন্ট ম্যাচিং সেটআপ

স্ক্যানিং ত্রুটি থাকা সত্ত্বেও সঠিকভাবে ছাত্রদের সনাক্ত করার জন্য এটি একটি গুরুত্বপূর্ণ বৈশিষ্ট্য।

কেন সেট আপ করবেন?

ছাত্রের প্রাথমিক আইডি (যেমন Student ID) অস্পষ্ট, অপাঠ্য বা ভুলভাবে পূরণ করা হলেও, তার রেকর্ডটি আপনার `Student Data Sheet` থেকে নির্ভরযোগ্যভাবে খুঁজে বের করতে।

এটি কীভাবে কাজ করে:

  1. Primary Match: আপনি একটি OMR শনাক্তকারী এবং একটি এক্সেল কলামের মধ্যে একটি প্রাথমিক লিঙ্ক সংজ্ঞায়িত করেন (যেমন, OMR-এর Student_ID ROI-কে আপনার এক্সেল ফাইলের `ID` কলামে ম্যাপ করুন)। অ্যাপ্লিকেশন সর্বদা প্রথমে এটি চেষ্টা করে।
  2. Secondary Matching (ফলব্যাক): প্রাথমিক ম্যাচ ব্যর্থ হলে, অ্যাপ্লিকেশনটি সেকেন্ডারি শনাক্তকারীর একটি সংমিশ্রণ ব্যবহার করে। উদাহরণস্বরূপ, আপনি সেট করতে পারেন যে যদি `Class` ROI `Class` কলামের সাথে এবং `Roll` ROI `Roll` কলামের সাথে মিলে যায় তবেই ম্যাচ হবে। যদি আপনার এক্সেল ফাইলের শুধুমাত্র একজন ছাত্র এই উভয় শর্ত পূরণ করে, তবে অ্যাপ্লিকেশনটি ভুল আইডি স্ক্যান সত্ত্বেও সঠিক ছাত্রকে খুঁজে পায়।

শনাক্তকারী রেফারেন্স

এই বৈশিষ্ট্যটি আপনার চূড়ান্ত প্রতিবেদনের জন্য কাঁচা স্ক্যান করা মানগুলিকে আরও পাঠযোগ্য পাঠ্যে ম্যাপ করে।

উদাহরণ:

আপনার OMR শীট 'Physics' বিষয়ের জন্য `02` কোড ব্যবহার করতে পারে। "Manage Identifier References" ডায়ালগে, আপনি `Subject_Code` ROI-এর জন্য একটি ম্যাপিং তৈরি করতে পারেন: "Scanned Value" `02` সেট করুন এবং "Used Value" `Physics` সেট করুন। এখন, আপনার সমস্ত আউটপুট এক্সেল ফাইলে অস্পষ্ট কোড '02'-এর পরিবর্তে পরিষ্কার নাম 'Physics' দেখাবে।

টেমপ্লেট বিল্ডার

টেমপ্লেট বিল্ডার হল যেখানে আপনি অ্যাপ্লিকেশনটিকে আপনার শারীরিক OMR শীটের লেআউট শেখান। এর আউটপুট হল একটি `.json` টেমপ্লেট ফাইল যা আপনার শীটের একটি ডিজিটাল মানচিত্র হিসাবে কাজ করে।

মেকানিজম ও কর্মপ্রবাহ

  1. Load Image: আপনার OMR শীটের একটি পরিষ্কার ছবি খুলতে Load Image ক্লিক করুন।
  2. Learn Corner Shape: 1. Learn Corner Shape ক্লিক করুন এবং আপনার শীটের চারটি বড় কোণার মার্কারের একটির চারপাশে একটি বক্স আঁকুন। অ্যাপ্লিকেশনটি এর বৈশিষ্ট্য বিশ্লেষণ করে অন্যগুলিকে স্বয়ংক্রিয়ভাবে খুঁজে বের করতে শেখে।
  3. Define Content Box: কোণ সনাক্ত করার পরে, 2. Define Content Box ক্লিক করুন এবং মূল বিষয়বস্তু এলাকার চারটি কোণায় ক্লিক করুন। অ্যাপ্লিকেশনটি একটি پرسپیکٹیو وار্প সম্পাদন করে, এই এলাকার একটি পুরোপুরি আয়তক্ষেত্রাকার, সোজা ছবি তৈরি করে।
  4. Draw ROIs (Regions of Interest): "Draw" টুল নির্বাচন করুন এবং যৌক্তিক বিভাগগুলির (ছাত্র আইডি ব্লক, উত্তর বুদবুদ ইত্যাদি) উপর আয়তক্ষেত্র আঁকুন।
  5. ROI বৈশিষ্ট্য নির্ধারণ করুন: একটি ROI নির্বাচন করুন এবং বাম দিকে তার বৈশিষ্ট্যগুলি নির্ধারণ করুন।
    • Name: একটি অনন্য নাম দিন (যেমন, `StudentID`)। এই নামটি নেমিং প্যাটার্নে প্লেসহোল্ডার হিসাবে ব্যবহৃত হয়।
    • Type:
      • `Identifier`: রোল নম্বরের মতো শনাক্তকারী মার্কারগুলির জন্য। বুদবুদের সারি/কলাম নির্দিষ্ট করুন। যে শনাক্তকারীগুলি সঠিক উত্তর কী খুঁজে বের করতে ব্যবহৃত হয় (যেমন `Exam_Code`), তার **Subtype** `Answer Script Identifier` হিসাবে সেট করুন।
      • `Answer`: উত্তর বিভাগগুলির জন্য। প্রশ্নের সংখ্যা (সারি), বিকল্প (কলাম), এবং প্রারম্ভিক প্রশ্ন নম্বর নির্ধারণ করুন।
  6. Save Template: আপনার লেআউট সংরক্ষণ করতে Save Template ক্লিক করুন।

উত্তর কী স্ক্যানার

উত্তর কী স্ক্যানার একটি সঠিকভাবে পূরণ করা OMR শীট থেকে একটি ডিজিটাল উত্তর কী তৈরি করে।

মেকানিজম ও কর্মপ্রবাহ

  1. Load Image: আপনার *সম্পূর্ণ* মাস্টার উত্তরপত্রের একটি স্ক্যান করা ছবি খুলতে Load Image ক্লিক করুন।
  2. স্বয়ংক্রিয় স্ক্যানিং: অ্যাপটি স্বয়ংক্রিয়ভাবে আপনার মাস্টার টেমপ্লেট (সেটিংস থেকে) ব্যবহার করে কোণ খুঁজে বের করে, ছবি ওয়ার্প করে এবং পূরণ করা বুদবুদগুলি পড়ে।
  3. Adjust and Re-Scan: স্ক্যান নিখুঁত না হলে, "Processing Controls" (কনট্রাস্ট, উজ্জ্বলতা ইত্যাদি) ব্যবহার করুন এবং নির্ভুলতা উন্নত করতে Re-Scan ক্লিক করুন।
  4. Verify: সনাক্ত করা উত্তরগুলি ডানদিকে প্রদর্শিত হয়। যেকোনো ভুল সংশোধন করতে চেকবক্সগুলিতে ম্যানুয়ালি ক্লিক করুন।
  5. Save Answer Key: Save Answer Key ক্লিক করুন। এটি টেমপ্লেট, চিত্র সেটিংস এবং সঠিক উত্তর সম্বলিত একটি `.json` ফাইল তৈরি করে।

উত্তর পরীক্ষক

উত্তর পরীক্ষক ছাত্র উত্তরপত্রগুলি ব্যাচ-প্রসেস করে, গ্রেড করে, ছাত্র ডেটার সাথে লিঙ্ক করে এবং একটি চূড়ান্ত প্রতিবেদন তৈরি করে।

স্ক্যানিং এবং ম্যাচিং মেকানিজম

  1. প্রাথমিক স্ক্যান: ছাত্রের শীট থেকে সমস্ত `Identifier` ROI-এর মান পড়ে।
  2. উত্তর কী ম্যাচিং: এটি `Answer Script Identifier` হিসাবে চিহ্নিত ROI-এর মানগুলি (যেমন, বিষয় কোড) ব্যবহার করে লোড করা সমস্ত উত্তর কীগুলির মধ্যে থেকে সঠিকটি খুঁজে বের করে। যদি কোনো কী না মেলে, স্ক্যানটি বিরতি দেয়।
  3. গ্রেডিং: ছাত্রের উত্তরগুলি মিলিত কী-এর সাথে তুলনা করে এবং একটি স্কোর গণনা করে।
  4. ছাত্র ডেটা ম্যাচিং: "Advanced Student Matching" নিয়ম ব্যবহার করে এক্সেল শীটে ছাত্রকে খুঁজে বের করার চেষ্টা করে, প্রথমে প্রাথমিক কী এবং তারপরে সেকেন্ডারি কী দিয়ে।
  5. ম্যানুয়াল হস্তক্ষেপ: যদি কোনো মিল অস্পষ্ট হয় (যেমন, একাধিক ছাত্র পাওয়া যায়), একটি ডায়ালগ আপনার ইনপুটের জন্য উপস্থিত হয়।
  6. সংরক্ষণ: চূড়ান্ত ফলাফল একটি এক্সেল ফাইলে যুক্ত করা হয়, যার ফাইলের নাম এবং কলামগুলি আপনার সক্রিয় "আউটপুট প্যাটার্ন" দ্বারা নির্ধারিত হয়।

কর্মপ্রবাহ

  1. Add Images: ছাত্র উত্তরপত্রের একটি ব্যাচ লোড করতে Add Images ক্লিক করুন।
  2. Select Answer Key(s): এক বা একাধিক উত্তর কী ফাইল লোড করুন। একাধিক লোড করলে, `Answer Script Identifier` প্রতিটি শীটের জন্য স্বয়ংক্রিয়ভাবে সঠিক কী নির্বাচন করতে ব্যবহৃত হবে।
  3. Start Scan: স্ক্যান শুরু করুন। ব্যাচ প্রসেসিংয়ের জন্য "Auto" মোড বা প্রতিটি শীট পর্যালোচনা করার জন্য "Manual" মোড ব্যবহার করুন।
  4. Handle Pauses: অস্পষ্ট মিল সমাধান করার মতো সাহায্যের প্রয়োজন হলে অটো মোডে স্ক্যানটি বিরতি দেবে।
  5. Review Results: স্ক্যান সম্পূর্ণ হওয়ার পরে, চূড়ান্ত ফলাফল দেখতে আপনার "Base Data Directory" থেকে তৈরি এক্সেল ফাইলটি খুলুন।
