OptiMark Pro User Manual

Introduction

Welcome to OptiMark Pro, a comprehensive OMR (Optical Mark Recognition) solution designed to streamline the process of creating, scanning, and evaluating OMR answer sheets. This manual provides a detailed guide to all the features of the software, from initial setup to advanced data processing.

Registration

The Registration module manages the activation of your software license. A valid license is required to unlock the full functionality of the application.

Functionality & Workflow

  1. Launch OptiMark Pro. If the license is invalid, most features will be disabled.
  2. Navigate to the Registration tab.
  3. Enter the license key you received into the input field.
  4. Click the Activate button. The application will connect to the license server to verify your key and lock it to your machine's unique hardware ID.
  5. If successful, all application features will be unlocked immediately.

Note: An active internet connection is required for license activation. The license is tied to a single machine.

Settings

The Settings page is the central hub for configuring the entire application. Proper setup here is crucial for all other modules to function correctly.

Default Paths

This section defines the core file locations for the application.

  • Base Data Directory: The main folder where all application data (templates, answer keys, results) will be stored.
  • Student Data Sheet: An Excel file containing student information. When you scan an answer sheet, the application uses this file to look up a student's details.
  • Master Template: The default OMR template to be used by the Answer Key Scanner. This is a critical setting, as the scanner needs a template to know the layout of the answer key sheet it is scanning.

File Naming & Output Patterns

This feature allows you to automate how your output files are named and organized. You can create multiple named patterns for different scenarios.

Example:

Imagine you want your final Excel reports to be named like Results_Class_IX_Physics.xlsx. In the "Setup" dialog, you can create a pattern for the "Excel Filename" by adding components in this order:

  1. Add a static text component: "Results"
  2. Add a component for the ROI named Class.
  3. Add a component for the ROI named Subject.

The application will now automatically use the values scanned from the "Class" and "Subject" ROIs on the sheet to generate this specific filename.

Advanced Student Matching Setup

This is a critical feature for accurately identifying students, even with scanning errors.

Why set it up?

To reliably find a student's record in your `Student Data Sheet` even if their primary ID is smudged, unreadable, or filled incorrectly.

How it works:

  1. Primary Match: You define a single, primary link between an OMR identifier and an Excel column (e.g., map the OMR's Student_ID ROI to the `ID` column in your Excel file). The application always tries this first.
  2. Secondary Matching (Fallback): If the primary match fails, the application uses a combination of secondary identifiers. For example, you can set it to match if `Class` ROI matches the `Class` column AND `Roll` ROI matches the `Roll` column. If only one student in your Excel file matches both criteria, the application finds the correct student, bypassing the unreadable ID.

Identifier References

This feature maps raw scanned values to more readable text for your final reports.

Example:

Your OMR sheet might use the number 02 for the subject 'Physics'. In the "Manage Identifier References" dialog, you can create a map for the `Subject_Code` ROI: set the "Scanned Value" to `02` and the "Used Value" to `Physics`. Now, all your output Excel files will show the clear name 'Physics' instead of the ambiguous code '02'.

Template Builder

The Template Builder is where you create a digital map (`.json` file) of your physical OMR sheet layout.

Mechanism & Workflow

  1. Load Image: Load a clean, unscanned image of your OMR sheet.
  2. Learn Corner Shape: Click 1. Learn Corner Shape and draw a box around one of the four large corner markers. The application analyzes its properties (size, shape) to find the others automatically.
  3. Define Content Box: Once corners are detected, click 2. Define Content Box and click the four corners of the main content area (e.g., the box containing all bubbles). The application performs a perspective warp, creating a perfectly rectangular, straightened image of this area. All subsequent work is done on this warped image.
  4. Draw ROIs (Regions of Interest): Select the "Draw" tool and draw rectangles over the logical sections (student ID block, answer bubbles, etc.).
  5. Define ROI Properties: Select an ROI and define its properties on the left.
    • Name: Give it a unique name (e.g., `StudentID`). This name is used as a placeholder in the Naming Patterns setup.
    • Type:
      • `Identifier`: For identifying markers like roll number. Specify rows/columns of bubbles. For identifiers used to find the right answer key (like `Exam_Code` or `Set`), set the **Subtype** to `Answer Script Identifier`.
      • `Answer`: For answer sections. Define the number of questions (rows), options (columns), and the starting question number.
  6. Save Template: Click Save Template to save your layout.

Answer Key Scanner

The Answer Key Scanner creates a digital answer key from a correctly filled-in OMR sheet.

Mechanism & Workflow

  1. Load Image: Load a scanned image of your *completed* master answer sheet.
  2. Automatic Scanning: The app automatically uses your Master Template (from Settings) to find corners, warp the image, and read the filled-in bubbles.
  3. Adjust and Re-Scan: If the scan is not perfect, use the "Processing Controls" (contrast, brightness, etc.) and click Re-Scan to improve accuracy.
  4. Verify: The detected answers appear on the right. Manually click checkboxes to correct any errors.
  5. Save Answer Key: Click Save Answer Key. This creates a `.json` file containing the template, ideal image settings, and the correct answers.

Answer Checker

The Answer Checker batch-processes student answer sheets, grades them, links them to student data, and generates a final report.

Scanning and Matching Mechanism

  1. Initial Scan: Reads all `Identifier` ROIs from the student's sheet.
  2. Answer Key Matching: It uses the values from ROIs marked as `Answer Script Identifier` (e.g., Subject Code) to find the correct answer key among all loaded keys. If no key matches, the scan pauses.
  3. Grading: Compares student answers to the matched key and calculates a score.
  4. Student Data Matching: Uses the "Advanced Student Matching" rules to find the student in your Excel sheet, trying the primary key first, then the secondary keys as a fallback.
  5. Manual Intervention: If a match is ambiguous (e.g., multiple students found), a dialog appears asking for your input.
  6. Saving: Appends the final result to an Excel file, with the filename and columns configured by your active "Output Pattern".

Workflow

  1. Add Images: Load a batch of student answer sheets.
  2. Select Answer Key(s): Load one or more answer keys. If you load multiple, the `Answer Script Identifier` will be used to automatically select the correct one for each sheet.
  3. Start Scan: Begin processing. Use "Auto" mode for batch processing or "Manual" mode to review each sheet.
  4. Handle Pauses: In Auto mode, the scan will pause if it needs your help to resolve an ambiguous match.
  5. Review Results: Open the generated Excel file from your "Base Data Directory" to view the final, graded results.
